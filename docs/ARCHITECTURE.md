### Error Handling & Fallback

```dart
Future<GenerateContentResponse> safeGenerateContent(
  GenerativeModel model,
  List<Content> content,
) async {
  try {
    return await model.generateContent(content);
  } catch (e) {
    if (_isModelNotFoundError(e)) {
      // Fallback to stable model
      final fallbackModel = GenerativeModel(
        model: 'gemini-1.5-flash',
        apiKey: apiKey,
      );
      return await fallbackModel.generateContent(content);
    }
    rethrow;
  }
}

  final msg = error.toString();
  return msg.contains('not found') || msg.contains('not supported');
}
```

### Data Access Control

```javascript
// Firestore Security Rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Users can only access their own data
    match /Farmers/{userId} {
      allow read, write: if request.auth != null 
                         && request.auth.uid == userId;
    }
    
    // Public read-only content
    match /content/{docId} {
      allow read: if request.auth != null;
      allow write: if false;  // Admin-only via Console
    }
  }
}
```

## Performance Considerations

### Optimization Strategies

1. **Lazy Loading**: Load data only when needed
2. **Caching**: Cache API responses locally
3. **Image Compression**: Compress images before upload
4. **Pagination**: Paginate large lists
5. **Widget Recycling**: Use ListView.builder for lists
